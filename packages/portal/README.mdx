# Portal

Portal renders its children into a separate subtree outside of the current component hierarchy. This is useful for overlays, modals, and any content that needs to appear above the rest of the UI.

```jsx
import { Portal } from 'startupjs-ui'
```

## Initialization

Wrap your application content with `Portal.Provider` so that portaled content has a host to render into.

```jsx
function Layout ({ children }) {
  return (
    <Portal.Provider>
      <Layout>{children}</Layout>
    </Portal.Provider>
  )
}
```

## Simple example

```jsx example
const [visible, setVisible] = useState(false)

const contentStyle = {
  position: 'absolute',
  top: 0,
  left: 0,
  width: u(25),
  height: u(20),
  backgroundColor: '#444',
  justifyContent: 'center',
  alignItems: 'center'
}

const textStyle = {
  color: '#fff',
  marginBottom: u(2),
  fontSize: u(3)
}

return (
  <Div>
    <Button onPress={()=> setVisible(true)}>Open</Button>
    {visible &&
      <Portal>
        <Div style={contentStyle}>
          <Span style={textStyle}>Content</Span>
          <Button
            variant='flat'
            onPress={()=> setVisible(false)}
          >Close</Button>
        </Div>
      </Portal>
    }
  </Div>
)
```

## Props

### Portal

- **children** -- the content to render into the portal host

### Portal.Provider

- **children** -- the application content that can use Portal
